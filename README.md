# Opt1 — private source repo

Closed-source macOS helper for *RuneScape* clue scrolls. The user-facing
README lives in [`JAHealey1/Opt1-Releases`](https://github.com/JAHealey1/Opt1-Releases);
this README is for future-me (or a contributor) and covers how the
release pipeline works.

## Repo layout

| Repo | Visibility | What lives there |
|---|---|---|
| `JAHealey1/opt1` (this) | Private | Source, tests, build scripts, GitHub Actions workflows, Sparkle integration. |
| `JAHealey1/Opt1-Releases` | Public | Notarized DMGs (as Release assets), `appcast.xml` (Sparkle feed), `release-notes/<version>.html`, third-party licence text. |
| `JAHealey1/homebrew-opt1` | Public | Homebrew tap. Single-file repo: `Casks/opt1.rb`, regenerated by CI on every release. |

## Concepts (so I don't have to relearn them in 12 months)

- **Apple Developer Program ($99/yr)** — entitles us to a Developer ID
  Application certificate. Without it, every macOS 14 user gets blocked
  by Gatekeeper on first launch. Enrolled to team **2KFN672Z9S**.
- **Hardened Runtime** — Xcode build setting (`ENABLE_HARDENED_RUNTIME = YES`)
  required by Apple's notary service. Locks down the process: no JIT, no
  unsigned-code injection, no library-validation bypass.
- **Entitlements file** ([`Opt1/Opt1.entitlements`](Opt1/Opt1.entitlements))
  — per-app exceptions to Hardened Runtime. Currently empty: Opt1 doesn't
  need any.
- **Notarization** — uploading the signed DMG to Apple, who scan it for
  malware and issue a *ticket* if it passes. Done via `xcrun notarytool
  submit --wait`.
- **Stapling** — gluing the notarization ticket onto the DMG with `xcrun
  stapler staple`, so first-launch works on a brand-new Mac with no
  internet.
- **Sparkle 2** — in-app updater. We integrate via Swift Package Manager
  (`https://github.com/sparkle-project/Sparkle`, 2.x). Provides the *Check
  for Updates…* UI, background polling, signed-download verification,
  install-on-quit, and relaunch.
- **EdDSA keypair** — Sparkle-specific signing key, separate from the
  Apple cert. Sparkle refuses to install any DMG whose EdDSA signature
  doesn't match the public key embedded in `Info.plist` (`SUPublicEDKey`).
  Defends installed users even if the public download server is
  compromised. **Do not rotate** — rotating breaks updates for everyone
  on an older Opt1 (they'd need to manually download the next release).
- **Appcast** — RSS feed at
  `https://raw.githubusercontent.com/JAHealey1/Opt1-Releases/main/appcast.xml`
  with one `<item>` per release. Sparkle polls this once a day; new
  `<item>` with a higher `sparkle:version` than what's installed triggers
  the update flow.
- **Homebrew Cask** — `auto_updates true`, `depends_on macos: ">= :sonoma"`.
  Tells brew that Sparkle handles updates so `brew upgrade` doesn't fight
  it. Bumps live in `Casks/opt1.rb` in the tap repo, regenerated from a
  template on every release.

## Cutting a release (the happy path)

```bash
# 1. Bump version in Opt1.xcodeproj/project.pbxproj.
#    Increment BOTH MARKETING_VERSION and CURRENT_PROJECT_VERSION on
#    BOTH Debug and Release configs of the Opt1 target. Sparkle compares
#    on CURRENT_PROJECT_VERSION (= CFBundleVersion at runtime), so it
#    MUST be a strictly larger integer than the prior shipped build.
#
# 2. Write release notes for this version. The release workflow REQUIRES
#    `release-notes/<version>.md` to exist and will fail the run if it
#    doesn't. See release-notes/README.md for the convention. The same
#    Markdown is rendered to HTML for Sparkle's "Check for Updates…"
#    dialog AND used as the GitHub Release body.

git add Opt1.xcodeproj/project.pbxproj release-notes/1.0.X.md
git commit -m "Bump to 1.0.X"
git push origin main

# 3. Tag and push. This is the entire release.
git tag v1.0.X
git push --tags
```

That triggers [`.github/workflows/release.yml`](.github/workflows/release.yml),
which does roughly:

1. Pin Xcode 26.3 on a `macos-26` runner.
2. Import the Developer ID Application cert into a temp keychain.
3. `xcodebuild archive` (Release config).
4. `xcodebuild -exportArchive` with `scripts/ExportOptions.plist`
   (`method = developer-id`).
5. `create-dmg` to produce `Opt1-X.Y.Z.dmg`.
6. `xcrun notarytool submit --wait` + `xcrun stapler staple`.
7. `spctl -a -vvv -t install` Gatekeeper sanity check.
8. Sparkle's `sign_update -f <key> Opt1-X.Y.Z.dmg` →
   captures `edSignature` + `length`.
9. Renders `release-notes/X.Y.Z.md` from this repo into HTML (via the
   Python `markdown` package) and copies it to
   `Opt1-Releases/release-notes/X.Y.Z.html` for Sparkle.
10. Appends a new `<item>` into `appcast.xml` via
    `scripts/update_appcast.py`, commits it alongside the rendered HTML
    notes to `Opt1-Releases`.
11. `gh release create vX.Y.Z` on `Opt1-Releases` with the DMG attached
    and the raw Markdown notes as the release body.
12. Renders `Casks/opt1.rb` from `scripts/Cask.opt1.rb.template` and
    pushes it to `homebrew-opt1`.

End users with Opt1 already installed pick up the update automatically
within ~24h via Sparkle, or instantly via *Opt1 → Check for Updates…*.
New users `brew install --cask opt1` or download the DMG directly.

## Manual fallback

If CI is broken (Apple notarization outage, expired secret, runner image
gone weird), [`scripts/release.sh`](scripts/release.sh) does steps 1–8
above on the dev Mac. Then push the resulting DMG, run
`scripts/update_appcast.py`, and create the release manually:

```bash
APPLE_ID=you@example.com \
APPLE_APP_SPECIFIC_PASSWORD=xxxx-xxxx-xxxx-xxxx \
./scripts/release.sh 1.0.X

# Inspect:
#   build/dist/Opt1-1.0.X.dmg
#   build/dist/release-1.0.X.json   (length / sha256 / edSignature)
#   build/dist/spctl-1.0.X.log      (Gatekeeper verdict)

# Push the appcast update against a local clone of Opt1-Releases:
python3 scripts/update_appcast.py 1.0.X \
    --build  Y \
    --length $(jq -r .length    build/dist/release-1.0.X.json) \
    --signature $(jq -r .edSignature build/dist/release-1.0.X.json) \
    --appcast /path/to/Opt1-Releases/appcast.xml

# Create the GitHub Release manually:
gh release create v1.0.X build/dist/Opt1-1.0.X.dmg \
    --repo JAHealey1/Opt1-Releases \
    --title "Opt1 1.0.X" \
    --notes-file release-notes-1.0.X.md
```

`scripts/release.sh` requires `create-dmg` (`brew install create-dmg`)
and the Developer ID Application cert + Sparkle EdDSA private key in
the login Keychain.

## CI on every push / PR

[`.github/workflows/ci.yml`](.github/workflows/ci.yml) builds Opt1 in
Debug (no code signing) and runs `xcodebuild test` on every push to
`main` and every PR targeting `main`. It's a quality gate so we only tag
releases off a known-green commit. Concurrency cancels superseded runs
on the same ref to keep the macOS-minute spend bounded.

## Troubleshooting common release failures

| Symptom | Likely cause | Fix |
|---|---|---|
| `"Opt1.app" is damaged and can't be opened` on a fresh Mac | Notarization ticket not stapled, OR the DMG was modified after stapling | Re-run notarize + staple. Don't edit the DMG after `xcrun stapler staple`. |
| Sparkle: `Update verification failed` | `SUPublicEDKey` in the installed app's Info.plist disagrees with the EdDSA private key used to sign the new DMG | Verify the private key in CI matches `t8SWpk74TGXrrnMstcQaM5ZAadD38/G9BNEmJwG+D4U=` (regenerate the secret from the login Keychain entry, not from a fresh keypair). |
| Sparkle says "You're up to date" but a new tag is live | `sparkle:version` (= `CFBundleVersion` = `CURRENT_PROJECT_VERSION`) didn't go up | Bump `CURRENT_PROJECT_VERSION` in **both** Debug and Release configs. Sparkle compares on integer build, not on semver string. |
| Hardened Runtime / notarization rejected | Build setting drift (e.g. `ENABLE_HARDENED_RUNTIME = NO` snuck back in) | Diff `Opt1.xcodeproj/project.pbxproj`. Both configs of the Opt1 target must have `ENABLE_HARDENED_RUNTIME = YES`. |
| `codesign` complains the embedded Sparkle.framework isn't signed | Old build artifact mixing in | `Product → Clean Build Folder` and `File → Packages → Reset Package Caches` in Xcode, then re-archive. |
| Workflow can't push to a public repo | `RELEASES_REPO_PAT` expired or scoped narrowly | Regenerate fine-grained PAT with `Contents: Read and write` on **both** `Opt1-Releases` and `homebrew-opt1`; update the repo secret. |
| Notarization stuck > 30 min | Apple notary backlog | Re-run the workflow. The `notarytool submit --wait` is idempotent; rerunning with the same DMG hash returns the cached verdict. |
| `spctl: rejected source=Notarized Developer ID` | Stapler ran on a path that wasn't the final DMG | Make sure stapling happens *after* `create-dmg`, not before. |

## Secrets reference

All seven secrets are repository-level on `JAHealey1/opt1`. Setup details
and rotation steps live in [`opt1_public_release_pipeline.plan.md` —
Appendix A](.cursor/plans/opt1_public_release_pipeline_cc816263.plan.md)
(local-only). Quick reference:

| Secret | Used by | Rotates? |
|---|---|---|
| `APPLE_ID` | `notarytool` | n/a — rotate the app-specific password instead |
| `APPLE_TEAM_ID` | `notarytool`, ExportOptions | n/a (constant: `2KFN672Z9S`) |
| `APPLE_APP_SPECIFIC_PASSWORD` | `notarytool` | revoke at appleid.apple.com → regenerate |
| `DEVELOPER_ID_CERT_P12_BASE64` | keychain import in CI | when cert nears 5-year expiry |
| `DEVELOPER_ID_CERT_PASSWORD` | keychain import in CI | in lockstep with the .p12 |
| `SPARKLE_ED_PRIVATE_KEY` | `sign_update` | **avoid rotating** — breaks updates for installed users |
| `RELEASES_REPO_PAT` | git push to `Opt1-Releases` and `homebrew-opt1`, `gh release create` | annually, before the PAT's 1-year expiry |

## Local development

Standard Xcode flow: open `Opt1.xcodeproj`, pick the Opt1 scheme, run.
On first launch macOS prompts for Accessibility (for the global hotkey)
and Screen Recording (for ScreenCaptureKit). Both are stored against the
bundle ID `com.jacobhealey.opt1`; if you ever rename the bundle ID,
`tccutil reset All com.jacobhealey.opt1` to clear stale TCC entries
before granting again.

The Python under `Opt1/Scripts/` is for offline ML/data work (puzzle ID
KNN, tile classifiers). It does not ship inside the `.app`.

## Release versioning rules

- Public-facing version is `MAJOR.MINOR.PATCH` (`MARKETING_VERSION`).
- Internal build number is a strictly-increasing integer
  (`CURRENT_PROJECT_VERSION`). Bump it on every shipped release, even
  patch-level, even if the semver doesn't change. Sparkle uses *only*
  this integer to decide "is there a newer version?".
- Tag format: `vMAJOR.MINOR.PATCH` (e.g. `v1.0.1`). The release workflow
  refuses any other tag shape.

## Out of scope (deliberately deferred)

- Submission to the official `Homebrew/homebrew-cask` tap.
- Custom domain for the appcast (currently `raw.githubusercontent.com`).
  Migration is one PR (bump `SUFeedURL` in `Info.plist`) and a DNS
  change.
- Sparkle delta updates and phased rollouts.
- Crash reporting / analytics.
- Self-hosted GitHub Actions runner. Only revisit if the 200 macOS-min
  free quota becomes a real bottleneck.
